Create my blog.

2020-03-29 Hugo

今天建立了blog，以下是对该过程的一个回顾与总结。

1 Git以及Hugo安装及配置

这一块在网上有很多教程，也没遇到什么难点，在此略过。

2 使用Hugo建立本地Blog

这一块主要参考Oh oh的博客进行建立，具体过程如下：

2.1 创建站点

首先创建站点：

# cd d:blog
hogo new site yourblogname
# Where 'yourblogname' is your blog name.

创建完成后，文件树如下：

├─archetypes
├─content
├─data
├─layouts
├─static
├─themes
└─config.toml

2.2 下载一个主题

在Hugo主题商店挑选一款自己喜欢的主题直接clone下来就行。

cd themes
git clone https://github.com/ertuil/erblog.git

themes文件夹用处存放各式主题，可以多下载几个尝试一下不同风格。

2.3 配置config.toml

这里我参考了两份官方文档，分别关于Configure与Taxonomies。 我的配置文件如下：

baseURL = "https://tesla-xiao.github.io/" # It seems ok without it.
title = "Tesla's blog"
theme = "erblog"
paginate = 8
languageCode = "zh-CN"
DefaultContentLanguage = "en"
enableInlineShortcodes = true
footnoteReturnLinkContents = "^"

[params]
    author = "Tesla"
    bio = """
    <p> Xiao Zhang is a Ph.D. candidate in the school of Statistics and 
        Management of SUFE. He majors in Applied Statistics, and his research 
        interests includes high dimensional statistics. </p>
    """
    copyright = "Copyright © 2020. All rights reserved."
    description = "Student in SUFE"

# ------------------------------------------------------------------------------

[[menu.main]]
    identifier = "zone"
    name = "Zone"
    url = "/zone/"
    weight = 2

# ------------------------------------------------------------------------------

[[menu.footer]]
    identifier = "Xu Liu's blog"
    name = "Xu Liu's blog"
    url = "https://xliusufe.gitee.io/post/hugomac/"
    weight = 1
    # My mentor's blog.

# ------------------------------------------------------------------------------

[taxonomies]
category = "categories"
tag = "tags"
series = "series"

# ------------------------------------------------------------------------------

[[menu.icon]]
    url = "mailto:zhangxiao_0422@163.com"
    name = "envelope-o"
    weight = 1

[[menu.icon]]
    url = "https://github.com/Tesla-Xiao"
    name = "github"
    weight = 2

2.4 创建一篇博客文章

博客的内容都存放在content文件夹下，我们可进一步划分文件夹post与zone来对应我们在博客上设置的板块。

首先创建文章文件hugo new post/build_blog.md。 然后可在md文件中进一步添加标签属性，如：

title: "Create my blog."
date: 2020-03-29T17:07:05+08:00
draft: false
tags: ["Hugo"]
# series: ["Some serious"]
categories: ["SKills"]

2.5 本地调试

回到根目录，使用 hugo server 命令调试预览。

cd ..
hugo server -t erblog --buildDrafts

其中 -t erblog 是指定主题为erblog，--buildDrafts 为编译草稿文件。

2.6 生成最终页面

cd ..
hugo -t erblog --buildDrafts

该过程会生成一个public文件夹，该文件夹的内容即Hugo生成的整个静态网站。

3 部署到Github及Gitee

3.1 关联本地库与Github及Gitee

由于我之前已经关联过了本地库与Github，所以只需关联本地库与Gitee即可。 关联本地库与Gitee，参考的是廖雪峰的教程。

3.1.1 上传SSH公钥至Gitee

上方教程有贴图详细教程，在此略过。

3.1.2 创建云仓库

分别在Github及Gitee创建相应的仓库。

注意，Github仓库名需设置为tesla-xiao.github.io，与我的用户名Tesla-Xiao相一致，方便之后使用 https://tesla-xiao.github.io 进行访问。

Gitee仓库名需设置为tesla-xiao，与我的用户名Tesla-Xiao相一致，方便之后使用 https://tesla-xiao.gitee.io 进行访问。

3.1.2 关联

具体操作如下：

cd public
git init
git remote add github git@github.com:Tesla-Xiao/tesla-xiao.github.io.git
git remote add gitee git@gitee.com:Tesla-Xiao/tesla-xiao.git
git remote -v # check remote library information

3.2 发布

# cd public
git add .
git commit -m "Create."
git push github master
git push gitee master

3.3 更新（或开通）pages服务（仅对于Gitee）

对于Gitee，需要购买Gitee Pages Pro服务才可自动部署。在发布后手动进行更新（或开通）流程如下：

首先到你的Gitee对应仓库，找到如下Gitee Pages图标并点击。

页面跳转后，再点击更新（或开通），等待部署完成。

接着就可以通过 https://tesla-xiao.gitee.io 访问你的主页了（可能实际更新存在一定的延时）。
